# orchestero-elements

A shadcn registry containing the current default `base-nova` components generated by `shadcn@4.13.0`.

The repo is a Next.js app so the built registry can be served from `public`, and it also works as a GitHub source registry through the root `registry.json`.

## What is included

- Next.js 16, React 19, Tailwind CSS v4, and shadcn `base-nova`.
- All default shadcn UI components from `npx shadcn@latest add --all`.
- AI chat app components under `components/chat`.
- The `use-mobile` hook used by the sidebar component.
- A generated source registry at `registry.json`.
- Built installable registry items in `public/*.json`.
- Mirrored registry items in `public/r/*.json` for direct app hosting.

## Install from this registry

Use the GitHub registry form:

```bash
npx shadcn@latest add khanhduyvt0101/orchestero-elements/button
npx shadcn@latest add khanhduyvt0101/orchestero-elements/sidebar
```

Or add a namespaced registry to a consuming app's `components.json`:

```json
{
  "registries": {
    "@orchestero": "https://cdn.jsdelivr.net/gh/khanhduyvt0101/orchestero-elements@main/public/{name}.json"
  }
}
```

Then install by namespace:

```bash
npx shadcn@latest add @orchestero/button
```

Install the AI chat component group:

```bash
npx shadcn@latest add @orchestero/chat-kit
```

Or install individual chat components:

```bash
npx shadcn@latest add @orchestero/chat-thread
npx shadcn@latest add @orchestero/chat-message
npx shadcn@latest add @orchestero/chat-composer
npx shadcn@latest add @orchestero/chat-tool-call
```

## Maintain the registry

Regenerate and build registry JSON after changing files in `components/ui` or `hooks`:

```bash
npm run registry:build
```

Validate the source manifest:

```bash
npm run registry:validate
```

Run app checks:

```bash
npm run typecheck
npm run lint
npm run build
```
